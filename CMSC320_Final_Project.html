Introduction

Polar Lows are a type of mesocyclone found in polar regions, oftentimes accompanied by heavy precipitation and strong winds. They can have a profound impact on coastal regions by disrupting fisheries, transportation, and tourism. Detection of Polar Lows can be challenging, so creating algorithm to classify them is crucial. Research shows that conversion of images to grayscale may result in more accurate classification when compared with their colored counterparts. Thus, the goal of our project is to determine which image conversions will result in the most correct classification of polar lows from supervised learning algorithms.

Conversion of images to grayscale has been shown to result in more accurate classification from a teachable machine when compared with their RGB counterparts (Bui et al, 2016). This is partly due to the fact that when images are converted to grayscale, there are less image features to focus on, making it easier to distinguish between different types of images.

Because Polar Lows are associated with severe weather, it is important to classify these correctly when determining current or upcoming atmospheric changes. If we are able to identify whether RGB or Grayscale images are better classified, this can be applied to future technologies that aim to detect and track Storm Clouds.

  4. Clouds without a spiral/comma shape OR not over a body of water

def seperate_images(list, dest_training, dest_testing, origin, amt):
  
  # Clears testing folder
  if len(os.listdir(dest_training)) > 0:
    for img in os.listdir(dest_training):
      os.remove(dest_training + "/" + img)

  # Clears training folder
  if len(os.listdir(dest_testing)) > 0:
    for img in os.listdir(dest_testing):
      os.remove(dest_testing + "/" + img)
    
  # Copies ALL images into its respective training set
  for img in list:
    sh.copy(origin + "/" + img, dest_training)

  # Helps to keep track of when 20% of the images are found
  i = 0

  # Copies 20% images out of the training set, into the testing set
  for img in os.listdir(dest_training):
    if(i != int(amt * .2)):
      sh.copy(origin + "/" + img, dest_testing)
      os.remove(dest_training + "/" + img)
      i = i + 1

def grayscale_image(path, dest):
    # Clears folder
  for img in os.listdir(dest):
    os.remove(dest + "/" + img)

  for img in os.listdir(path):
    curr_img = path + "/" + img

    img_grey = Image.open(curr_img).convert('LA')
    
    # Saves the image as greyscale
    img_grey.save(dest +"/" + img)

Mean Squared Error:

Let's take a look at the Mean Squared Error. For images, MSE works to indicate perfect similarity. The Mean Squared Error between the two images is the sum of the squared difference between the two images. The lower the error, the more "similar" the two images are.

Below we will use this technique to compare an RGB image to its exact same grayscale counterpart. By doing this, we want to see how similar these images actually are in order to help hypothesize how our Supervised Learning Algorithm will perform.

Because each image in the RGB and Grayscale folder have the same file names, this made the process of taking the MSE easy to iterate through. Additionally, we will do this evaluation on our testing set for simplicity.

def mse(imageA, imageB):
  # Reads in image based on file path
  imageA = skimage.io.imread(imageA)
  imageB = skimage.io.imread(imageB)

  # Calculates the error using both images sum of squared differences
  err = np.sum((imageA.astype("float") - imageB.astype("float"))**2)
  err /= float(imageA.shape[0] * imageA.shape[1])
	
  # Adds error number to array
  similarity_arr.append(err)

def alter_res(curr, dest):

  # Deletes images currently within that folder
  for img in os.listdir(dest):
    os.remove(dest + "/" + img)

  # Itereates through current test pictures and blurs them using CV2
  for img in os.listdir(curr):

    # Reads in image file
    image = cv2.imread(curr + "/" + img)

    # Blurs the image
    image = cv2.blur(image, (10,10))

    # Saves it to its new directory
    cv2.imwrite(dest + "/" + img, image)

Conclusion and Results

The goal of our project was to determine if converting satellite images into grayscale would make them easier to classify in the eyes of our teachable machine models. Teachable machine is a basic wrapper around google's tensorflow api, including a user interface. We trained two models, one with RGB images and another with grayscale images. We trained the model with 80% of our dataset, witholding 20% for testing.

After testing our model and displaying the results in confusion matrices, we noticed there was not a large difference in results between the model trained on RGB images and grayscale images. The RGB model correctly classified all 28 polar low images, misclassified 20 non polar lows as polar lows, and only misclassified 2 non satellite images. The grayscale model also correctly classified all 28 polar low images, but misclassified one more non polar low than the RGB model.

The results of testing our models with blurred images was expectedly worse than normal images. This shows positive results for our model, as lower resolution images should result in worse classification. The biggest difference after the resolution change was more non polar lows were misclassified as polar lows.

The expected result of our project was that the grayscale model would classify images with a higher accuracy than that of a model trained on RGB images. This was actually not the case, with our RGB model outperforming the grayscale model by a single image. Such similar accuracies does not provide a definitive answer to our question of if a grayscale model would perform better. The largest reason for this discrepancy is in the data we had available to us. Much much more data would be needed to provide an accurate training of teachable machines. This combined with the fact that our data was hand picked by students and may not have been accurately labeled or separated could have lead to some discrepancies in the models decision making.
